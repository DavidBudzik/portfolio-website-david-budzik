Designing a dual-mode interface that makes pattern matching accessible to non-technical users. A progressive disclosure approach that brings regex power to compliance officers and security administrators managing AI governance policies.

The Observation

A Quote That Became the North Star

"I'm scared I'll break something, so I just add 47 individual domains."

— Compliance Officer, Financial Services Company

During a customer support session at Ovalix, I watched a compliance officer at a financial services company configure an AI data sharing policy. She was controlling which email domains could share data with ChatGPT—a high-stakes AI governance task where wrong configuration could expose sensitive data to AI services.

She knew exactly what she needed (regex would be perfect), but the existing tools forced an impossible choice between tedious repetitive entry and complex pattern matching. This quote became the foundation for everything that followed.

Problem Space

The Gap Between Simple and Complex

The problem wasn't lack of features—it was the absence of middle ground. Users felt inadequate, not empowered. They built workarounds rather than use the feature. In AI governance, this gap was more than an annoyance—it was a security and compliance blocker.

Simple Lists

Easy to use but painfully repetitive. Users manually enter dozens of individual items (email addresses, domain names, AI tool names) when patterns would work better. No way to express "all company domains" or "detect credit card numbers."

Pattern Matching

Powerful but intimidating. Regex syntax errors surface at runtime, creating anxiety in high-stakes compliance environments. No safety net, no guidance, no templates—just a blank field and fear of exposing sensitive data.

01

Context Switching Overhead

Mental model disruption in high-stakes AI governance environments. Users couldn't smoothly transition between simple and complex without feeling lost or overwhelmed.

02

Regex Intimidation

Not just syntax anxiety—fear of compliance violations and data exposure. Healthcare teams needed HIPAA patterns, finance needed PII detection, all needed confidence.

03

No Safety Net

Runtime errors could mean compliance violations. Validation happened too late, with no preview or testing capability. Users deployed policies without confidence they would work.

04

Import Friction

Teams resorted to spreadsheets for AI governance rules. Bulk operations lacked intelligence and transparency, making migration from legacy systems painful and error-prone.

Reframing

What If It Was One Tool That Grew With You?

The breakthrough came from stopping to think of these as separate features. Instead of "Basic Tool" and "Advanced Tool," what if they were points on a spectrum? Progressive disclosure became the key principle.

The toggle isn't a switcher—it's an amplifier. The same mental model, different depths. Complexity becomes opt-in, not mandatory. Make the transition feel natural, like turning up the volume rather than changing the channel.

The Solution

String Mode: Simple by Default

The default experience requires zero learning curve. Open the modal, see a single text field and one button. Type what you want to add, hit Add, and it appears in the table. Success in under 30 seconds.

No configuration, no choices to make. Just the familiar pattern of adding items to a list. This is "simple by default" in action—meeting users exactly where they are.

String mode: one text field, one button, zero learning curve

Powerful by Choice

Regex Mode: Visible, Not Scary

When users toggle to Regex mode, they don't face a blank field. Instead, curated templates appear—each showing syntax, description, and type. This isn't just shortcuts; it's teaching through use.

Templates cover 80% of use cases: email patterns, domain wildcards, SSN detection, credit card patterns, date formats. Users see regex with context, making the powerful accessible without dumbing it down.

Template Library

Curated patterns for common AI governance scenarios. Each template shows the regex syntax alongside a plain-language description. Users learn by selecting, not by studying documentation.

Real-Time Validation

Immediate feedback as you type. Syntax errors appear instantly with specific messages, not generic warnings. Build confidence to experiment because you know immediately if you made a mistake.

Template dropdown showing syntax + description + type

Real-time validation with specific error messages

Unified View

Mixed Lists & Table Interface

Users don't think in pure categories. They need both simple entries AND patterns in the same list. The table view supports this coexistence with type badges that make scanning effortless.

Inline editing reduces cognitive load—no mode switching required. Click to edit, changes save immediately. The interface respects existing mental models while adding sophisticated capabilities.

Table view with type indicators and inline editing

Bulk Operations

Smart Import with Transparency

Bulk operations are critical for migrating from legacy AI governance systems. The import flow has intelligence—it infers types from patterns automatically—but maintains transparency through an editable preview state.

Users review and adjust before committing. No mysterious magic, just smart defaults with full control. Trust comes from seeing what will happen before it happens.

Import flow: upload → preview → review → commit

Design Principles

Framework for Every Decision

Simple by Default, Powerful by Choice

Start with one text field. Advanced features are always one toggle away, but never forced. Complexity is opt-in, not mandatory.

Make Regex Visible, Not Scary

Show syntax with context. Templates educate through use. Real-time validation eliminates anxiety. Power without intimidation.

Feedback Before Commitment

Validate, preview, confirm. Users see results before they're permanent. Build confidence through transparency at every step.

Respect Existing Mental Models

Standard patterns, expected behaviors. Tables for lists, toggles for modes, inline editing for quick changes. Familiar patterns reduce friction.

Reflection

What I Learned

What Worked

Progressive disclosure turned complexity into confidence. Template library became educational content, not just shortcuts. Real-time validation eliminated the biggest source of anxiety. Users felt smarter, not dumber.

What I'd Do Differently

Add pattern testing feature earlier. Invest more in import intelligence for edge cases. Plan for scale from the start—large lists need search and filter. Consider natural language input carefully (powerful but needs guardrails).

"Good design isn't about hiding complexity—it's about revealing it at the right moment, with the right support. Sometimes the best interface makes users feel smarter than they did before."

— Key Learning

Context beats documentation. Users learned regex through templates and immediate feedback, not manuals. Trust comes from transparency—seeing what will happen before it happens. Constraints (synchronous validation) became features (instant feedback).

Make complexity opt-in. The toggle isn't a switcher; it's an amplifier. Start simple, grow sophisticated. Same mental model, different depths. This approach works across domains—from AI governance to any interface bridging novice and expert users.
