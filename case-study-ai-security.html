AI Agent Security Platform

Designing the first comprehensive B2B platform for monitoring and securing AI agents and Model Context Protocol (MCP) connections in enterprise environments.

Project Overview

Fast MVPs Without Sacrificing Design Quality

I work on a cybersecurity platform that protects and monitors AI agents and MCP (Model Context Protocol) connections in organizations. In security, you need fast feedback loops—getting features in front of security teams quickly to learn what actually matters before committing full engineering resources.

But as a product designer, I also needed to maintain visual consistency and design system integrity. The challenge: deliver testable MVPs in one week instead of three, while meeting the design quality standards that make user feedback meaningful.

The Challenge

Browser-based AI tools generated inconsistent code—random colors instead of design tokens, spacing that violated our grid, components that ignored our patterns. The "time savings" evaporated when I had to rewrite everything to meet design standards.

The Breakthrough

Stopped using natural language and started using structured JSON specifications with exact design token references. Built a workflow connecting Claude, Figma Make, and Cursor through MCP to maintain consistency at every step.

Design Process

The AI-Native Workflow

Foundation

Design Token Infrastructure

Mapped every Figma variable to its CSS token with exact syntax. Example: Figma color/grey/200 → CSS --igds-color-grey-200: #e8e9e9. This became the foundation everything depends on—no arbitrary values, only token references.

Step 01

Structured JSON Specifications

Built a Claude skill that generates comprehensive JSON prompts from wireframes. Instead of natural language descriptions, I created structured specifications with component types, exact design token references, measurements, state variations, and accessibility requirements. JSON forces precision—AI stops guessing and starts executing exactly.

Step 02

Figma Make for Mockups

Using the JSON specifications, Figma Make generates design-system-connected mockups that properly reference design tokens. Every color, spacing value, and typography choice comes from the token system—ensuring pixel-perfect fidelity to design standards before any code is written.

Step 03

Cursor with MCP Integration

Through Figma MCP, Cursor accesses design context directly, pulls Code Connect mappings, and composes components into our codebase with version management. The tools aren't working in isolation—they're connected to our design system and codebase through MCP. Context is everything.

Validation

Client Feedback Loop

Deploy MVPs to select clients within one week. First feedback round reveals what monitoring features security teams actually need. Iterate using the same workflow—JSON updates flow through to code. Second round confirms we're solving real problems before committing full engineering resources.

Key Features

What We Built

01

Agent Inventory Dashboard

Real-time monitoring of all AI agents in the organization. Security teams can see agent status, permissions, recent actions, and risk levels at a glance. Every element uses design tokens—consistent spacing, proper typography scale, and systematic color usage.

02

MCP Connection Tracking

Visual representation of Model Context Protocol connections showing what data agents can access and which external services they interact with. Makes invisible relationships visible while maintaining design system integrity.

03

Agent Detail Sidebar

Deep-dive interface showing agent identity, permissions, connection history, and behavior patterns. Designed for quick investigation without losing context. Built using the same JSON specifications that ensured consistency throughout.

04

Security Alerts System

Intelligent alerting for suspicious agent behavior, unauthorized access attempts, and policy violations. Alert hierarchy uses systematic color tokens to communicate severity without overwhelming users.

Design Decisions

What Actually Mattered

Structure beats speed: Browser tools were faster to start but produced unusable output—random values, inconsistent spacing, ignored design systems. The structured workflow took more setup but delivered testable MVPs that enabled meaningful user feedback.

Design quality as acceptance criteria: As a product designer, I couldn't compromise on visual consistency or design system integrity. MVPs need proper token usage, grid alignment, typography scale, and accessibility to represent your product accurately. Speed without design quality wastes user testing time.

JSON specifications enable consistency: Natural language produces guesses. JSON with design token references produces predictable implementations that meet design standards. When you give AI structured data instead of prose, it stops guessing and starts executing precisely.

Integration over isolation: AI tools connected to our design system through MCP dramatically outperformed isolated code generators. Tools working in isolation can't maintain consistency. Tools connected to your design system and codebase can.

Multiple specialized tools, orchestrated workflow: Claude Skills for data structure, Figma Make for visuals, Cursor with MCP for integration. Each tool specialized for its purpose, the workflow orchestrating them into a coherent system that maintains design quality at every step.
