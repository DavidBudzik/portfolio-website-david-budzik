Walkthrough Experience for Able

Designing an interactive onboarding experience for a research and decision intelligence platform, helping users discover value through personalized use-case flows and guided first project creation.

Project Context

The Onboarding Challenge

Able Insights is a research and decision intelligence platform that connects private data, proprietary datasets, and public sources to help corporate development teams, market researchers, and investment analysts make informed decisions. The platform offers access to 50+ data sources, AI-powered insights, and collaborative research workflows.

The challenge was creating an onboarding experience that could serve three distinct user personas—corporate development professionals, market researchers, and investment analysts—each with different needs, workflows, and levels of technical expertise. Users needed to quickly understand the platform's value and start their first research project without getting lost in features.

The Problem

New users faced analysis paralysis when presented with 50+ data sources and multiple use cases. They needed guidance on where to start based on their specific role and goals, not a generic product tour.

The Goal

Create a personalized walkthrough that adapts to the user's role and use case, guiding them to create their first research project while introducing only the features relevant to their workflow.

Design Principles

Guiding Philosophy

Show, don't just tell

Instead of explaining features abstractly, the walkthrough demonstrates them with real examples. Users see the product in action rather than reading descriptions of what it can do.

Progressive disclosure

Information is revealed as users need it, not all at once. Basic concepts come first, advanced features later. Users can skip ahead if they're already familiar with concepts.

Respect user agency

The walkthrough is helpful, never prescriptive. Users can exit anytime, skip steps, or revisit sections. They're in control of their learning journey, not trapped in a rigid tutorial.

Quick wins matter

Within the first 90 seconds, users accomplish something meaningful. This early success builds confidence and momentum for exploring more complex features later.

The Solution

A Multi-Layer Approach

Rather than a single linear walkthrough, I designed a layered education system that adapts to different user needs and learning styles. The experience has three modes that work together seamlessly.

Layer 1: Essential Walkthrough

A focused 2-minute introduction that covers only the core workflow. Users complete one meaningful task and understand the product's basic mental model. This is the default experience for new users.

Screen 1: A confident wrap-up with quick access to core resources.

Screen 2: CTA-led momentum into the first workflow build.

Screen 3: Automation benefits framed with measurable outcomes.

Screen 4: Personalised folders that reinforce long-term organisation.

Layer 2: Contextual Tooltips

As users explore, subtle tooltips appear when they encounter new features for the first time. These are minimal, dismissible, and only show when contextually relevant. They don't interrupt workflow.

Layer 3: Resource Center

A persistent help menu where users can watch specific feature tutorials, read best practices, or revisit the initial walkthrough. This serves users who prefer to explore first and learn later.

Interaction Design

Making Education Feel Natural

Spotlight highlighting: Rather than covering the interface with overlays, the walkthrough uses subtle dimming to highlight relevant areas. Users can still see the full interface, maintaining context while focusing attention.

Interactive demonstrations: Users don't just watch—they perform actions themselves with guidance. This hands-on approach creates muscle memory and builds confidence faster than passive observation.

Smart pacing: The system detects when users are struggling and offers additional help. If users are moving confidently, it steps back. Pacing adapts to individual needs rather than following a fixed timeline.

Graceful exits: At any point, users can exit the walkthrough and return to normal use. A small indicator shows how far they've progressed, and they can resume later if desired. There's no penalty for skipping.

User satisfaction with onboarding

The new walkthrough significantly improved user activation and retention. More importantly, qualitative feedback showed that users felt more confident exploring the product on their own after completing the onboarding.

The layered approach proved particularly successful. Power users appreciated being able to skip the basic walkthrough, while novice users benefited from the structured guidance. By respecting different learning styles and experience levels, the design served a broader range of users effectively.

Key Learnings

What This Project Taught Me

Onboarding is not a single moment: The best user education happens across multiple touchpoints. A single comprehensive tutorial overwhelms; small, contextual lessons throughout the experience work better.

Completion rate isn't everything: We optimized for user confidence and long-term activation, not just walkthrough completion. Some users who skip the tutorial entirely become highly engaged—and that's okay.

Test with real tasks, not toy examples: Early prototypes used generic demo content. When we switched to realistic use cases, engagement improved dramatically. Users need to see themselves in the examples.

Respect attention as a finite resource: Every second in an onboarding flow has a cost. We ruthlessly prioritized what users truly needed to know immediately versus what could wait until they needed it.
